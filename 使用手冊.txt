	git status(查看現在自己電腦上的修改狀態)
	git add 檔案名稱 （把你剛剛修改的東西變成歷程儲存）
	git add .(記得要空格再打點，把所有修改的東西變成歷程儲存)
	git commit -m “打上你想描述的東西”（你剛剛的歷程儲存需要描述後才能真正修改儲存）
	git push origin main (推送到github讓大家看到，這個步驟才是真的大家看得到，重要！！)
    git pull origin main （把雲端上目前的最新工作內容更新到你自己的電腦中）
    git log 用來查看專案的提交歷史，顯示各次提交（commit）的詳細資訊，例如提交的哈希值、作者、提交日期和提交訊息。這個指令可以讓你了解專案的變更紀錄，方便追溯修改的內容與原因。
